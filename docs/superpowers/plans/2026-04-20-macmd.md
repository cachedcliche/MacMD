# Plan: Simple macOS Markdown Editor

## Context

The user wants a clean, secure, TextEdit-style desktop app for writing and saving `.md` files. It is for their boss and will be reviewed by ChatGPT, so the bar is: functions perfectly, clean UI, smooth, and free of obvious security/footgun issues. Based on clarifications:

- **Scope:** Plain text editor with **live syntax highlighting** (no split preview pane).
- **Distribution:** Local `.app` bundle (no App Store submission).
- **Minimum OS:** macOS 14 Sonoma.

Greenfield project — no existing code to modify.

## Approach

A single-target **SwiftUI document-based macOS app** built with Xcode. SwiftUI's `DocumentGroup` + `FileDocument` provide File → New/Open/Save/Save As/Recent/autosave/versions for free, which is exactly the TextEdit-like behavior the user wants with essentially zero custom code. The text surface is an `NSTextView` wrapped via `NSViewRepresentable` (SwiftUI's `TextEditor` does not support per-range attributes, so it cannot do syntax highlighting cleanly). Syntax styling is applied by an `NSTextStorage` delegate that re-highlights only the paragraph ranges touched by each edit, so typing stays smooth in long documents.

**App name:** `MacMD` (used everywhere — display name, Xcode project, target, Swift module, folder name). **Bundle ID:** `com.eb.MacMD`.

## Key implementation details

### 1. `MarkdownDocument.swift`
- Conforms to `FileDocument`.
- `readableContentTypes = [.init(filenameExtension: "md")!, .plainText]`.
- Stores `String`. Reads/writes UTF-8 (validate on read; fall back to `String(decoding:as:)` with `UTF8.self` to avoid crashes on bad bytes).
- `fileWrapper(configuration:)` returns a `FileWrapper` with UTF-8 data.

### 2. `MacMDApp.swift`
```swift
@main
struct MacMDApp: App {
    var body: some Scene {
        DocumentGroup(newDocument: MarkdownDocument()) { file in
            DocumentView(document: file.$document)
        }
        .commands { /* minimal; rely on defaults */ }
    }
}
```
This single declaration yields File → New/Open/Save/Save As/Duplicate/Revert/Recent + autosave + versions.

### 3. `MarkdownTextView.swift` (`NSViewRepresentable`)
- Wraps `NSScrollView` + `NSTextView` (non-rich-text, `isRichText = false`, `allowsUndo = true`, `importsGraphics = false`, `smartInsertDeleteEnabled = false`).
- Monospaced font (`NSFont.monospacedSystemFont(ofSize: 14, weight: .regular)`).
- Generous padding via `textContainerInset = NSSize(width: 24, height: 20)`.
- Bound to document text via SwiftUI `@Binding<String>`.
- Installs `MarkdownHighlighter` as `textStorage.delegate`.

### 4. `MarkdownHighlighter.swift`
- Implements `NSTextStorageDelegate.textStorage(_:didProcessEditing:range:changeInLength:)`.
- Expands the edited range to its **paragraph range** (`(storage.string as NSString).paragraphRange(for:)`) and re-applies attributes there only — fast on long docs.
- Rules (compiled `NSRegularExpression` per pattern, stored statically):
  - ATX headings: `^(#{1,6})\s.+$` → bold, size step per level, accent color.
  - Bold: `\*\*[^*\n]+\*\*` and `__[^_\n]+__` → bold.
  - Italic: `(?<!\*)\*[^*\n]+\*(?!\*)` and `_[^_\n]+_` → italic.
  - Inline code: `` `[^`\n]+` `` → monospaced + subtle background via `.backgroundColor`.
  - Fenced code block: paragraph-anchored lines between ```` ``` ```` → monospace, muted color.
  - Links: `\[([^\]]+)\]\(([^)]+)\)` → underlined accent color on label, muted on URL.
  - Unordered list marker: `^\s*[-*+]\s` → accent color on marker only.
  - Ordered list marker: `^\s*\d+\.\s` → accent color on marker only.
  - Blockquote: `^\s*>.*$` → italic, muted color.
  - HR: `^\s*(-{3,}|\*{3,}|_{3,})\s*$` → muted.
- Colors come from `Theme.swift` using **semantic `NSColor`** (e.g. `.labelColor`, `.secondaryLabelColor`, `.linkColor`, `.controlAccentColor`) so Dark Mode works automatically.
- Always resets `.font`, `.foregroundColor`, `.backgroundColor`, `.underlineStyle` on the range first before re-applying, so deletions clear old styling.

### 5. `DocumentView.swift`
- Pure SwiftUI wrapper around `MarkdownTextView`. No toolbar, no sidebar — the window is the text.
- Sets reasonable default window size: `.frame(minWidth: 520, minHeight: 400, idealWidth: 760, idealHeight: 680)`.

### 6. Entitlements & security
- `com.apple.security.app-sandbox = true`
- `com.apple.security.files.user-selected.read-write = true`
- No network, no camera/mic, no Apple Events.
- Hardened runtime on (Xcode default for Release).
- Ad-hoc sign (`Signing: Sign to Run Locally`) is sufficient for a boss-reviewed local build; this still opens on any Mac via right-click → Open on first launch.

### 7. Info.plist highlights
- `CFBundleName = MacMD`, `CFBundleDisplayName = MacMD`.
- `CFBundleDocumentTypes`: one entry declaring `.md` as owner/editor, role Editor, `LSHandlerRank` Alternate.
- `UTExportedTypeDeclarations`: export `net.daringfireball.markdown` conforming to `public.plain-text` with extension `md`.
- `LSApplicationCategoryType = public.app-category.productivity`.

### 8. Tests (`MacMDTests/MarkdownHighlighterTests.swift`)
- Unit tests against the regexes: heading levels 1–6, bold/italic nesting edge cases, inline code, fenced blocks, links, list markers, blockquote, HR.
- Each test asserts that applying the highlighter to a sample `NSTextStorage` produces the expected attributes at known ranges.

## Build, run, and verify

From Xcode:

1. `open /Users/eb/MacMD/MacMD.xcodeproj`
2. Select scheme **MacMD**, target **My Mac**, hit ⌘R.
3. Run tests with ⌘U (`MacMDTests` target).

From CLI (headless):

```
xcodebuild -project /Users/eb/MacMD/MacMD.xcodeproj \
  -scheme MacMD \
  -configuration Release \
  -destination 'platform=macOS' \
  build
```

The built `.app` will be under `~/Library/Developer/Xcode/DerivedData/MacMD-*/Build/Products/Release/MacMD.app`.

## Manual smoke test (end-to-end)

1. Launch → a new untitled document opens. Window is clean, monospaced, padded.
2. Type a heading (`# Hello`), bold (`**bold**`), italic, code (`` `x` ``), a link, and a fenced code block → each styles live as typed.
3. Copy/paste a paragraph from another app → pastes as plain text (no rich attributes leak in).
4. ⌘S → Save dialog offers `.md` by default → saves to Desktop.
5. Close the window, reopen via File → Open Recent → content round-trips exactly (byte-for-byte).
6. Toggle System Settings → Appearance → Dark → colors adapt automatically.
7. Open a 5 MB markdown file → typing stays smooth (scoped-highlight guarantees no full-doc reflow per keystroke).
8. Force-quit mid-edit → on relaunch, autosaved version is restored (DocumentGroup default).

## What this plan deliberately does not include

- No live rendered preview pane (user chose syntax highlighting only).
- No App Store packaging (user chose local `.app`).
- No custom toolbar, sidebar, theming UI, word count, or export-to-HTML. The brief is "simple like TextEdit" — adding these would violate the spec.
- No CloudKit/iCloud sync — out of scope.
